Please rebuild the local ChatGPT Fast client for a non-official API integration.

Why: when ChatGPT/Codex uses a non-official API, compatibility endpoint, or proxy gateway, the original client usually hides or downgrades Fast mode. Rebuild the official Microsoft Store OpenAI.Codex app locally so the Fast selector and service-tier request path are available.

Run:

powershell -ExecutionPolicy Bypass -File ".\refresh-chatgpt-fast.ps1" -ForceClose

Confirm that the script:

1. Copies the newest Microsoft Store OpenAI.Codex app.
2. Extracts and repacks resources\app.asar.
3. Patches Speed visibility, featureRequirements.fast_mode permission, and the request-side Fast gate.
4. Adds Fast/priority service tiers to the model catalog when applicable.
5. Sets service_tier = "fast" in config.toml after creating a backup.
6. Preserves UserData and the previous portable build.
7. Starts ChatGPT Fast and verifies Settings > General > Speed contains Standard and Fast.

Finally, inspect the non-official API gateway log. The gateway must accept and forward service_tier/priority; a client-side patch cannot add Fast capability to a backend that ignores those fields.
